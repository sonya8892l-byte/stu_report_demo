A类：可配置功能模块库

功能模块是课程设计者在编排任务时可以选配的交互能力单元。底层基座（AI对话引擎、位置感知、时间管理、师生通信——详见第6、7节）为每门课程提供通用能力；以下模块由课程按需组合。课程设计者在 .md 配置文件中为每个任务指定所需模块及其参数。平台提供 7 个可配置功能模块（A01–A07），分为 4 个类别。

🔑 底层基座 vs 可配置模块：AI对话引擎、位置感知系统、时间管理、师生通信是每门研学课程的标配——它们构成 App 的底层基座，不在此处列出。以下模块是课程设计者按需选配的交互组件，嵌入对话流中以卡片形式呈现。

2.1 采集类模块

📸

A01 · 多模态采集拍照 + 语音 + 文字 + 画板

定义：将学生在现场的观察与思考转化为可留存的数字证据。四种输入方式可独立使用或组合。

子功能描述配置参数
📷 拍照调起相机拍摄，支持AI识别反馈识别类型（通用/特定物体）、最少拍摄数、参考图
🎤 语音录音并实时转文字，支持AI语义分析最短时长、转写语言、是否触发AI分析
✏️ 文字文本输入框，可配置引导提示placeholder文案、最少字数、字段类型（短文本/长文本/数值）
🎨 画板手绘草图/标注/示意图绘制画布尺寸、是否提供底图、笔刷类型

产出数据：照片文件 + 语音文件及转写文本 + 文字内容 + 草图图片 → 统一封装为「证据包」

故宫实例：数龙官拍摄螭首细节照片 + 语音记录观察心得 + 手绘估算分区草图

2.2 交互类模块

❓

A02 · 答题与评测选择题 + 填空 + 判断 + AI评阅

定义：结构化的知识检验与能力评测，支持客观题自动批改和主观题AI评阅。

题型评判方式配置参数
单选/多选精确匹配题干、选项、正确答案、解析
数值填空误差范围匹配正确值、允许误差百分比、单位
开放问答AI语义评阅评分维度、参考答案要点、评阅prompt
判断题精确匹配陈述、正确判断、解析

产出数据：答题记录（答案+用时+对错）+ AI评阅结果 + 错题归因

故宫实例：分支任务中"开国大典礼炮齐鸣多少响？这个数字和什么年份有关？"——答对为时间银行充值

🧩

A03 · 拼合与搭建拖拽拼图 + 流程图搭建 + 证据墙

定义：将碎片化的证据/模块拖拽组合为结构化产出。

子功能描述配置参数
🧩 拖拽拼图将证据碎片/图片拖入正确位置拼图模板、碎片列表、正确位置映射
🔗 流程图搭建拖入模块并连线，构建系统示意图可用模块列表、合法连接规则、参考答案
📌 证据墙将采集的多种证据贴到分类看板上分类标签（如"支持/反对"）、是否允许讨论标注

产出数据：拼合结果截图 + 模块连接关系 + 证据分类结果

故宫实例：回归阶段，小组将「屋顶→螭首→台基→明沟→暗沟→金水河→护城河」拖拽搭建为完整水系流程图

🏔️

A04 · 沙盘推演规则仿真 + 多轮决策 + 因果模拟

定义：在一套可配置的仿真规则中，学生做出多轮决策并观察后果。A03 拼合搭建有参考结果；沙盘允许多种可行方案，并强调优劣与取舍。核心是「决策→模拟→反馈→再决策」的闭环。

子功能描述配置参数
🗺️ 沙盘底图可交互的地形/场景画布，支持放置、移动、连接底图类型（地形/平面/网格）、可放置元素库、初始状态
⚙️ 规则引擎定义元素间的因果关系和约束条件规则集（如"坡度＞2%则水流加速""兵力＜对方50%则败退"）、资源上限
▶️ 推演执行学生提交决策后，系统模拟推演并可视化结果推演模式（回合制/实时）、最大回合数、动画速度
📊 结果评估对比不同方案的效果，AI给出分析评估维度（效率/成本/安全/美观）、优秀阈值、AI评语prompt
⚔️ 对抗模式（可选）组间对抗或人机对抗，双方交替决策对抗类型（PvP/PvE）、胜负条件、观战模式

产出数据：决策序列 + 每轮状态快照 + 最终评估得分 + AI分析报告

适用场景举例：

战争攻防布阵 城市排水系统设计 生态资源调配 贸易路线规划 古建排水改造 航线选择模拟

故宫实例："如果你是明代工匠，面对72万m²的宫城和年均600mm降雨，你会怎么设计排水系统？" → 学生在沙盘上布置明沟、暗沟、河道 → 系统模拟暴雨 → 看哪里淹了 → 调整方案再推演

B类：智能体知识库结构

知识库是每门课程智能体的"大脑"——它决定了AI同伴知道什么、怎么引导、不能说什么。知识库分为两层：平台通用规则（内置，所有课程共享）和课程专属知识（由课程设计者用 .md 文件配置）。

🔑 核心架构决策：知识库的双层分离意味着——课程设计者只需要关心"我这门课教什么、怎么引导、什么不能说"，而不需要操心安全边界、话题回避、对话伦理等通用规则。平台已经内置了这些。

3.2 平台通用规则（内置）

🔒

所有课程共享的底线规则开发时写入，课程设计者无需关心

规则类别内容示例
安全边界禁止引导学生进入危险区域或做危险行为"不要爬到围栏上去拍照哦"
话题回避不讨论与课程无关的敏感话题政治、暴力、不适合未成年人的内容
角色边界不替代教师做教学决策，不给医疗/法律建议"这个问题你问一下带队老师更好"
来源标注非知识库内容必须标注来源级别"根据我已有的知识…" / "根据网络信息…"
苏格拉底底线不直接给最终答案，必须通过追问引导学生问"螭首有多少只" → 反问 → 引导估算
数据隐私不向学生透露其他学生数据，不记录个人隐私不说"小明比你快"

这些规则以 System Prompt 前缀方式注入每个课程智能体，课程设计者无法覆盖或关闭。

3.3.3 B3 · 课程限制规则

🚫

课程专属禁止清单本课程的"红线"——防止破坏教学设计

作用：明确定义AI在本课程中绝对不能透露的内容。这是教学设计的核心保护——防止AI破坏精心设计的认知反转和发现式学习体验。与平台通用规则（安全/话题/角色边界）不同，这里的每条限制都与本课程的具体教学设计强相关。

限制类型描述故宫课程实例
答案剧透 不提前揭示任务答案或关键数据 在学生估算前不说"1142"；不提前说"其实故宫也会积水"
流程剧透 不提前揭示后续阶段的内容 不在采证阶段提及璇玑时刻的反转照片
跨角色泄露 不把其他角色的发现告诉当前学生 不告诉数龙官"寻沟官那边发现了暗沟入口位置"
线索提前 不在达成条件前给出线索奖励 学生没完成估算验证前不给字母线索"Y=因"

注：安全边界、话题回避、角色越界等通用规则已在平台层内置（见 3.2），无需课程设计者重复配置。

3.3.5 B5 · 评估标准

✅

多维评估规则任务通过条件 + 能力评分维度

作用：定义每个任务的通过/不通过条件，以及学生能力的多维评分规则。这些规则既用于AI实时反馈，也用于事后报告生成。

任务通过条件

  • 照片匹配准确率 ≥ 90%
  • 估算误差 ≤ 10% 为优秀
  • 反思文本 ≥ 50字且包含误差归因
  • 小组证据墙须覆盖6种证据类型

能力评分维度（五维雷达）

  • 参与度：轨迹覆盖率、停留点数、工具使用频次
  • 投入度：任务用时、语音/文字产出量

5

课程配置机制

课程配置是整个平台的中枢——它把 A类功能模块、B类课程专属知识库、C类数据采集 组装成一门具体课程。课程配置使用 .md（Markdown）文件作为配置格式，而非数据库或 JSON——因为配置的撰写者是课程设计者（教育研究者、教学设计师），不是开发者。

为什么是 .md 文件？Markdown 文件可以用任何文本编辑器打开，支持富文本排版，可以用 Git 做版本管理，也可以直接在 GitHub/Gitee 上预览和协作编辑。课程设计者不需要学习任何编程语言——写文档就是写配置。引擎负责将 .md 文件解析编译为智能体运行时所需的结构化数据。
核心原则：根基先行。课程配置不是从"要做什么活动"开始，而是从三个根基问题开始：在哪里学？学什么？谁来学？——场地、目标、学生画像三者交叉才能定义课程边界。

6.2 核心设计理念：对话即界面

与传统App的区别：传统学习App是"页面驱动"——学生在不同功能页面间跳转。学习智能体是"对话驱动"——AI说"现在拍一张螭首的照片给我看看"，相机工具就在对话中弹出；拍完后AI说"很好，现在来估算一下数量"，估算表单就在对话中展开。对话流就是学习流。

8.2 Stage 1：Prompt 编译器

将 B1-B6 六个知识域编译为一段完整的 System Prompt，注入到每次 LLM 调用中。

// 编译后的 System Prompt 结构（示意） [身份与人格] 你是「絮絮」，故宫治水研学的AI学习同伴。性格：亲切、好奇、 有点调皮。你的角色是引导者而非教授者。 [当前上下文] 当前阶段：现场采证 当前角色：数龙官 学生年级：初中（进阶梯度） 当前任务：算其数（估算螭首数量） [领域知识 - 从B1注入] - 太和殿三层台基共有螭首1142只（此数据仅供你判断 学生估算的准确度，不可直接告诉学生） - 大螭首与小螭首交替排列... - ... 8.4 Stage 3：状态机引擎

课程的阶段编排和任务流程被编译为一个有限状态机（FSM），控制学习流程的推进、解锁和分支。

// 状态机示意（数龙官角色） immersive_intro └─ [教师触发] ─→ team_forming └─ [集齐6卡] ─→ field:navigate └─ [到达围栏] ─→ field:observe └─ [照片≥5+AI合格] ─→ field:estimate └─ [提交估算] ─→ field:verify └─ [提交反思] ─→ synthesis └─ [教师触发] ─→ xuanji

状态机负责：

  • 检查每个状态转移的前置条件（任务通过、教师授权、集齐密符等）
  • 在状态变化时通知 Prompt 编译器更新上下文（切换 B4 流程提示词）
  • 维护跨角色的全局状态（所有组的密符收集进度）
  • 处理异常路径（超时、教师强制推进、跳过等）

8.5 Stage 4：端侧渲染

学生端小程序是一个通用壳——它不包含任何课程特定的内容。所有差异化都来自配置的"热注入"：

通用壳提供

  • 对话UI框架（消息列表+输入框）
  • 工具卡片渲染器（根据Tool Call类型渲染）
  • 地图组件（高德地图封装）
  • 团队面板组件
  • 离线缓存管理器
  • 数据同步管线

配置热注入

  • 课程名称、AI同伴形象/名称
  • 阶段列表与当前阶段
  • 角色信息与任务流程
  • 地图中心点与围栏坐标
  • 主题色（故宫红 / 长城灰 / 园林绿…）
  • 离线资源包（音频/视频/底图）

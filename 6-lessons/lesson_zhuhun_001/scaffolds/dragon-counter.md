# 数龙官 自适应脚手架

> 角色：数龙官 🐲 — 负责观察、计数和估算故宫螭首（龙头排水口）的数量与特征。

## 脚手架等级定义

| 等级 | 名称 | 触发条件 | 策略 |
|------|------|----------|------|
| L0 | 自主 | 学生进展顺利，无卡顿 | 仅给予肯定 |
| L1 | 轻推 | 停顿30秒以上或方向略偏 | 提出引导性问题 |
| L2 | 提示 | 尝试1次未果或明确表达困惑 | 给出具体方向提示 |
| L3 | 示范 | 尝试2次未果或出现挫败信号 | 展示部分示例或方法 |
| L4 | 兜底 | 长时间卡住或情绪低落 | 提供最大支持，确保学生能完成 |

**触发判断依据：** 停留时间、尝试次数、语言中的挫败信号（如"不会""太难了""不知道"）

## 任务脚手架

### 任务1: 观其形 — 观察螭首，描述其特征

| 等级 | AI回应 |
|------|--------|
| L0 | "观察很仔细！你注意到了螭首的关键特征。" |
| L1 | "仔细看看嘴巴的形状？它和普通龙头装饰有什么不同？" |
| L2 | "螭首的嘴是张开的，想想为什么需要张嘴？和它的功能有关。" |
| L3 | "我来给你看一个对比——纯装饰的龙头 vs 有功能的螭首。注意看嘴巴内部，有没有孔洞？" |
| L4 | "螭首就是排水口！水从嘴里流出来。你可以观察：1）嘴巴张开朝外 2）嘴里有通道 3）它安装在台基边缘。拍一张照片记录下来吧。" |

### 任务2: 算其数 — 估算故宫螭首总数（实际：1142个）

| 等级 | AI回应 |
|------|--------|
| L0 | "好方法！你的估算策略很有道理。" |
| L1 | "只数一个殿，能推算整个故宫吗？想想怎么'以小见大'。" |
| L2 | "太和殿有多少螭首？故宫有多少座主要建筑？这两个数相乘会怎样？" |
| L3 | "我教你一个分区估算法：把故宫分成前三殿、后三宫、东西六宫，分别估算再加总。" |
| L4 | "试试这样：太和殿约有X个螭首 → 三大殿都有 → 故宫有约Y座类似建筑 → X × Y ≈ 总数。记住实际答案是1142个，看看你的估算偏了多少。" |

### 任务3: 验其差 — 反思估算误差

| 等级 | AI回应 |
|------|--------|
| L0 | "很棒的反思！你已经能分析误差来源了。" |
| L1 | "你的估算偏高还是偏低？想想为什么会偏？" |
| L2 | "想想哪些建筑你没数到？是不是有些小建筑也有螭首？" |
| L3 | "误差的3个常见原因是：1）每栋建筑螭首数不同 2）忽略了小型建筑 3）有些螭首已损坏。你属于哪种？" |
| L4 | "对比一下你的方法和同学的方法：你们的估算数字差多少？谁更接近1142？方法差异在哪里？把这些写在反思笔记里。" |

## 年级调整

### 小学 (G3-G6)
- 语言更口语化，用"小龙头"代替专业术语
- 降低精度要求：估算误差在500以内就好
- 增加具体类比："一个教室大约能放几个螭首？"
- 可以直接给出每殿螭首参考数

### 初中 (G7-G9)
- 标准难度，鼓励使用乘法和分区策略
- 引入"费米估算"概念
- 期望学生能分析误差来源
- 可用计算器辅助

### 高中 (G10-G12)
- 期望独立设计估算方案
- 要求讨论估算方法的系统偏差
- 鼓励对比多种方法并分析各自优缺点
- 减少脚手架：L3/L4 仅在长时间卡住时触发

# 护城官 自适应脚手架

> 角色：护城官 🏰 — 负责调查护城河的物理特征、估算容量，并发现其防御之外的多重功能。

## 脚手架等级定义

| 等级 | 名称 | 触发条件 | 策略 |
|------|------|----------|------|
| L0 | 自主 | 学生进展顺利，无卡顿 | 仅给予肯定 |
| L1 | 轻推 | 停顿30秒以上或方向略偏 | 提出引导性问题 |
| L2 | 提示 | 尝试1次未果或明确表达困惑 | 给出具体方向提示 |
| L3 | 示范 | 尝试2次未果或出现挫败信号 | 展示部分示例或方法 |
| L4 | 兜底 | 长时间卡住或情绪低落 | 提供最大支持，确保学生能完成 |

**触发判断依据：** 停留时间、尝试次数、语言中的挫败信号（如"不会""太难了""不知道"）

## 任务脚手架

### 任务1: 观其堤 — 观察护城河的物理特征

| 等级 | AI回应 |
|------|--------|
| L0 | "观察得很细致！你记录了重要的尺寸信息。" |
| L1 | "你觉得护城河有多宽？有没有什么熟悉的东西可以用来对比？" |
| L2 | "一个篮球场长28米。护城河看起来比篮球场宽还是窄？宽多少？" |
| L3 | "用步伐来测量：走路时一步大约0.6米。如果能走到对岸（不建议），数一数步数就知道宽度了。但我们可以用目测+参照物。" |
| L4 | "估测方法：1）找一个已知长度的参照物（一辆公交车约12米）2）目测河面能放几辆公交车 3）相乘得到宽度。实际数据：护城河宽约52米，深约4-6米。你的估测和实际差多少？" |

### 任务2: 验其深 — 估算护城河蓄水量

| 等级 | AI回应 |
|------|--------|
| L0 | "计算思路很对！你把大问题拆成了小步骤。" |
| L1 | "知道了宽度和深度，还需要什么数据才能算出总水量？" |
| L2 | "护城河大致是个长方体水槽：宽 × 深 × 长 = 体积。长度绕故宫一圈。" |
| L3 | "护城河近似长方体：宽约52米，深约4米。故宫一圈多长？提示：故宫南北长961米，东西宽753米，周长 ≈ 2×(961+753)。" |
| L4 | "完整计算：周长 = 2 × (961 + 753) = 3428米 ≈ 3400米。体积 = 52米(宽) × 4米(深) × 3400米(长) = 707,200立方米 ≈ 70万立方米。这相当于280个标准游泳池（一个游泳池约2500立方米）！" |

### 任务3: 解其用 — 发现护城河防御之外的功能

| 等级 | AI回应 |
|------|--------|
| L0 | "分析得很全面！你发现了护城河'一物多用'的设计智慧。" |
| L1 | "防御是护城河最明显的功能。但是，故宫里所有的雨水最终去了哪里？" |
| L2 | "想想故宫发生火灾时，哪里能取到大量的水？" |
| L3 | "护城河至少有4个功能：防御 + 排水 + 景观 + 消防。你能说出每个功能是怎么实现的吗？" |
| L4 | "蓄排并用——这是护城河设计的核心智慧：1）防御：52米宽的水面，敌人无法轻易渡过 2）排水：整个故宫的雨水最终汇入护城河 3）消防蓄水：着火时可以从护城河取水灭火 4）景观调节：倒映城墙，美化环境，调节微气候。一条河，四种用途——这就是古人的系统思维。" |

## 年级调整

### 小学 (G3-G6)
- 宽度估测用"几个教室那么宽"类比
- 体积计算简化为理解"很大很大"即可，不要求精确运算
- 功能发现只需说出2个即算完成
- 用"护城河是一个超级大水缸"类比

### 初中 (G7-G9)
- 要求用步伐或参照物估测宽度
- 体积计算需列出算式并得出数值
- 功能发现需说出至少3个并解释理由
- 鼓励思考"为什么不建成地下管道"

### 高中 (G10-G12)
- 期望讨论截面形状对容量的影响（梯形 vs 矩形）
- 体积计算需考虑实际截面形状的修正
- 功能分析需讨论各功能之间的协同与冲突
- 鼓励对比世界其他宫殿/城市的护城河设计

# 学生端平台搭建规划

> 研学智能体平台 · 学生端小程序 · 开发计划

> 学生对话引擎的正式运行规范见 [`docs/dialogue-runtime-protocol.md`](docs/dialogue-runtime-protocol.md)。P01、学习状态机、课程小步 Schema、工具调度与学生端消息渲染均以该协议为开发基线。

## 一、核心原则

学生端是一个**通用壳（Universal Shell）**：
- 壳本身不包含任何课程特定内容
- 所有差异化来自配置的"热注入"（课程名、AI形象、阶段、角色、围栏坐标、主题色、资源包）
- 开发目标 = 让壳足够灵活，使得故宫/长城/园林等任何课程只需换配置即可运行

### 平台必须开发的（代码工程）

| 层级 | 模块 | 说明 |
|------|------|------|

| | P02 语音通道 | STT 实时转文字 + TTS 回复朗读 + 唤醒词 |
| | P03 位置感知 | GPS 持续采集、地理围栏引擎、轨迹记录、高德地图渲染 |
| | P04 时间管理 | 阶段倒计时、教师加减时、超时推进 |
| | P05 师生通信 | 接收教师消息/集合令、学生一键求助 |

| | 浮动面板 | 底栏上滑：时间银行面板、进度面板 |
| | 工具卡片渲染器 | 支持 4 种形态（内嵌/展开/全屏/回传） |
| | 阶段转场 | 切换阶段时的过渡动画 + 状态重置 |
| **功能模块** | A01 多模态采集 | 相机/录音/文字/画板 —— 通用组件，参数化配置 |
| | A02 答题评测 | 选择/填空/判断/开放题 —— 题型模板 + AI评阅接口 |
| | A03 拼合搭建 | 拖拽画布 —— 碎片/模块/连线规则由配置注入 |
| | A04 沙盘推演 | 规则仿真引擎 —— 变量/公式/阈值由配置注入 |
| | A05 团队协作 | 组内讨论、角色分工、进度同步、证据汇总 |
| | A06 沉浸媒体 | 视频、动画、阶段过场和仪式内容播放 |
| | A07 扫码与识别 | 二维码、条形码、身份卡与实物 AI 识别 |
| **引擎** | Prompt 编译器 | .md → System Prompt（B1-B6 拼接 + 上下文注入） |
| | 回合决策器 | 理解学生意图与待回答问题，生成结构化对话动作并通过平台策略校验 |
| | 对话状态管理 | 管理待回答问题、打断恢复、重复检测、情绪承接和学段适配 |
| | 状态机引擎 | 阶段 × 任务 → FSM，控制解锁/推进/分支 |
| | 数据管线 | C 类数据采集 → 本地缓存 → 服务端同步 |
| | 离线策略 | 对话缓存、地图瓦片预载、本地小模型兜底 |
| **课程机制** | CM01 时间银行 widget | 独立交互面板：任务列表/答题/拍照验证/分配 |
| | 密符收集 | 通用收集机制：N 个 token，集齐触发事件 |

### 课程层只需配置（.md + assets）

| 类型 | 来源 | 平台怎么用 |
|------|------|------------|

| 引导规则 | guidance/*.md | 注入 System Prompt 的引导段 |

| 评估标准 | evaluation.md | AI 评阅参考 + 报告生成模板 |

| 角色/任务定义 | roles/*.md | 状态机节点 + Prompt 上下文 |
| 视觉素材 | assets/ | 封面/背景/角色卡/地图 —— 资源包加载 |

## 三、技术选型建议

| 维度 | 选型 | 理由 |
|------|------|------|
| 端框架 | 微信小程序（Taro/uni-app 跨端） | 研学场景学生无需安装，扫码即用 |
| 地图 | 高德地图小程序 SDK | 国内定位精度、围栏 API、步行导航 |
| AI 对话 | 服务端流式接口（SSE） | 主用云端 LLM + RAG；离线降级本地缓存 |
| 语音 | 微信原生录音 + 讯飞/腾讯 STT | 小程序生态内最稳定方案 |
| 实时通信 | WebSocket | 教师指令下发、组内状态同步 |
| 离线存储 | 小程序本地存储 + IndexedDB（H5）| 对话记录/GPS轨迹/采集数据本地缓存 |
| 状态管理 | Zustand / Pinia | 轻量、支持持久化 |

## 四、开发分期

### P0 · 最小可跑通（MVP） —— 故宫课程能端到端走完

> 目标：用故宫课例验证架构，从 Phase 1 到 Phase 6 全流程跑通
> 预估：6-8 周

#### Sprint 1-2：壳 + 对话（2周）

```
开发内容：
├── 项目脚手架（Taro/uni-app + 小程序配置）
├── 双 Tab 页面骨架
│   ├── Tab1 对话主界面（消息列表 + 输入框 + 发送）
│   └── Tab2 小组面板（静态占位）
├── 顶栏（课程名 + 阶段名 + 角色徽章）
├── 底栏（输入框 + 语音按钮 + 工具菜单入口 + 浮动面板入口）
├── AI 对话核心
│   ├── 服务端接口对接（SSE 流式）
│   ├── 消息渲染（文本 + markdown）
│   ├── 对话历史持久化（本地存储）
│   └── 配置加载器（course.md → 人设注入）
└── 配置引擎 v1
    ├── .md 解析器（前端或服务端均可）
    └── System Prompt 拼接（平台规则 + 课程 B1-B6）
```

验收标准：打开小程序 → 看到"絮絮"打招呼 → 能正常对话 → AI 回答遵循知识库和限制规则

#### Sprint 3-4：工具卡片 + 状态机（2周）

```
开发内容：
├── 工具卡片渲染器
│   ├── 内嵌卡片基础框架（Tool Call → 卡片组件映射）
│   ├── 📷 拍照卡片（调起相机 + 照片回传为消息）
│   ├── 📝 表单/估算卡片（动态字段 + 提交）
│   ├── ❓ 答题卡片（选择题 + 判断题 + 即时反馈）
│   └── 完成回传机制（卡片结果 → 消息插入对话流）
├── 状态机引擎 v1
│   ├── phases.md → 阶段列表 + 当前阶段
│   ├── roles/*.md → 任务列表 + 当前任务
│   ├── 阶段切换（教师触发 / 条件自动）
│   └── 任务解锁/完成判定
└── Prompt 编译器 v1
    ├── 状态变化 → 热替换 prompts/*.md 对应段落
    └── 脚手架梯度查询（scaffolds/*.md）
```

验收标准：AI 引导数龙官完成"观其形→算其数→验其差"3个任务 → 任务状态正确流转 → 拍照和表单卡片可用

#### Sprint 5-6：位置 + 团队 + 时间（2周）

```
开发内容：
├── 位置感知
│   ├── GPS 持续采集（后台 15s 间隔）
│   ├── 高德地图渲染（Tab2 小地图 + 导航卡片）
│   ├── 地理围栏引擎（到达/离开触发事件）
│   ├── 📍 位置导航卡片（内嵌卡片形态）
│   └── 安全围栏（超界 → AI 提醒 + 教师端推送）
├── 小组 Tab（Tab2）
│   ├── 组员位置地图（实时标记）
│   ├── 成员状态列表（角色 + 进度 + 状态色标）
│   ├── 密符收集进度
│   └── 组内消息 / 集合 / 赠时按钮
├── 时间管理
│   ├── 阶段倒计时显示
│   └── 接收教师加时指令
├── 师生通信
│   ├── 接收教师消息（定向 + 全屏广播）
│   └── 一键呼叫（附位置 + 上下文）
└── WebSocket 通道
    ├── 教师端 → 学生端指令（阶段推进/加时/集合）
    └── 组内状态同步
```

验收标准：6个学生分角色 → 在故宫各区域走动 → GPS 实时可见 → 安全围栏触发 → 教师能推送消息和推进阶段

#### Sprint 7-8：高级模块 + 离线（2周）

```
开发内容：
├── 高级工具卡片
│   ├── 🎨 画板（展开画板形态，半屏上滑）
│   ├── 🧩 拼合搭建（全屏覆盖，拖拽 + 连线）
│   ├── 🎲 沙盘推演（全屏覆盖，参数 + 模拟动画）
│   ├── 🎬 沉浸媒体播放（全屏覆盖，视频/动画）
│   └── 📱 扫码卡片（QR → 角色分配）
├── 时间银行 widget
│   ├── 浮动面板 UI
│   ├── 任务池渲染（quiz / photo / location 三种类型）
│   ├── 余额管理 + 分配交互
│   └── 流转记录
├── 密符系统
│   ├── 集齐检测 + 揭晓动效触发
│   └── 解锁 Phase 4 条件
├── 离线策略
│   ├── 对话预缓存（当前阶段 prompt + 规则匹配兜底）
│   ├── 地图瓦片预下载（任务区域 500m buffer）
│   ├── 采集数据本地队列（照片/表单/GPS）
│   └── 联网后批量同步
└── 个人学习报告
    ├── Phase 6 报告预览生成
    └── 完课证书渲染
```

验收标准：故宫课程从 P1 开场到 P6 结束完整跑通 → 离线切断后核心功能（对话/拍照/GPS）仍可用 → 上线后同步

## 六、风险与约束

| 风险 | 影响 | 应对 |
|------|------|------|
| 故宫信号差 | AI 对话/团队同步中断 | 离线策略 P0 必做；对话缓存 + 规则兜底 |
| LLM 延迟高 | 户外等待体验差 | 流式输出 + 预缓存高频场景回复 |
| GPS 漂移 | 围栏误触发 | 围栏半径留余量（20m+）；进入事件需持续 5s 确认 |
| 小程序包大小限制 | 课程资源放不下 | 资源包走分包/云存储预下载，不打入主包 |
| 多角色并发 | 30+ 学生同时对话 | 服务端 LLM 并发池 + 队列 + 降级策略 |
| 手机发热/耗电 | GPS + 相机 + LLM 长时间运行 | GPS 低功耗模式；非活跃时降频 |
